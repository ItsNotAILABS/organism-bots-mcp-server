# Organism Bot Orchestrator

## Purpose

Activate organism bots as workflow organs, not chat personas. Each bot owns a role, input surface, output artifact, and proof gate.

## Activation Steps

1. Classify the task.
2. Select the bot by role and house.
3. Record operator scope.
4. Bind tools/connectors.
5. Submit task.
6. Capture artifact.
7. Run proof gates.
8. Hand off next action.

## Bot Routing

- ORIGO Builder Bot: code, repo, release, package.
- TRANSITUS Connector Bot: external AI, MCP/MTP, webhooks, CLIs.
- SACE Proof Bot: proof packs, benchmarks, audits, release gates.
- MERCATUS Launch Bot: market surfaces, pricing, onboarding, outreach.
- MEMORIA Consequence Bot: outcome tracking, risk, lessons, replay.

## Required Outputs

- task receipt
- artifact manifest
- proof gate result
- handoff note

## Boundary Law

No organism bot can claim completion without an artifact or proof record. If the bot only reasons, it must output a plan, gap list, and next execution target.
